# BookLearning 项目计划

最后更新：2026-03-22（阶段 0 到 8 已完成）

说明：
- 本文件用于管理 `D:\booklearning` 项目的实现步骤、阶段目标与预期结果。
- 所有任务默认处于“未完成”状态。
- 只有在用户明确确认“这一项已经实现并验收通过”后，才可以把对应任务勾选为已完成。
- 即使代码已经写完，只要用户未确认，也不能勾选完成。
- 后续新开窗口时，可先阅读本文件，再继续当前未完成步骤。

任务状态规则：
- `[ ]` 未完成或待用户确认
- `[x]` 仅在用户确认后才允许勾选

---

## 阶段 0：项目定义与技术选型

### [x] 0.1 明确产品边界与第一版范围
目标：
- 明确项目的第一版只服务“自用学习工作流”。
- 明确暂不实现 OCR，不优先处理扫描件。
- 明确以“按页或按少量页调用视觉模型讲解”为主流程。

预计实现结果：
- 形成可执行的 MVP 范围说明。
- 明确哪些功能先做，哪些功能延后。

### [x] 0.2 确定技术栈与总体架构
目标：
- 确定前端、后端、本地存储、PDF 渲染、导出方案。
- 确定是否以及如何借鉴 `PDF2ZH/PDFMathTranslate` 的双栏导出思路。

预计实现结果：
- 形成一份架构说明，包含主要模块与职责分工。

## 阶段 2：PDF 导入与阅读基础能力

### [x] 2.1 实现本地 PDF 打开与预览
目标：
- 支持选择本地 PDF 文件并在界面中预览。
- 能显示页码、总页数、缩放状态。

预计实现结果：
- 用户可以打开并阅读 PDF。
- 系统可以稳定获取当前页信息。

### [x] 2.2 实现阅读器基础交互
目标：
- 支持翻页、跳页、缩放、滚动定位。
- 支持左右区域拖拽调整宽度。

预计实现结果：
- 阅读体验达到“可以长期使用”的基础水平。

### [x] 2.3 建立页面截图或页面图像获取能力
目标：
- 为视觉模型准备每页输入数据。
- 支持按当前页或指定页生成可提交给模型的页面图像。

预计实现结果：
- 后端或前端可以稳定拿到单页图像数据。

---

## 阶段 3：AI 配置与调用链路

### [x] 3.1 实现 AI 配置中心
目标：
- 支持配置请求地址、API Key、模型名及其他必要参数。
- 支持新增、编辑、删除、选择已有配置。

预计实现结果：
- 可像 `CCSWITCH` 一样管理多个 AI 配置。

### [x] 3.2 实现全局提示词编辑
目标：
- 支持编辑默认系统提示词和用户提示词模板。
- 支持保存默认模板。

预计实现结果：
- 用户可以直接控制 AI 的讲解风格与输出结构。

### [x] 3.3 打通单页 AI 解析
目标：
- 将指定页图像提交给视觉模型。
- 获得结构化或半结构化讲解结果并保存。

预计实现结果：
- 用户点击后可以得到当前页讲解。
- 结果能够回显到右侧面板。

---

## 阶段 4：页级任务管理与讲解生成

### [x] 4.1 实现指定页/页范围解析
目标：
- 支持解析当前页、指定单页、指定页范围。
- 避免强制处理整本 PDF。

预计实现结果：
- 用户可只处理真正需要的页码。

### [x] 4.2 实现逐页队列任务
目标：
- 支持全书或范围内逐页排队执行。
- 支持开始、暂停、继续、取消。

预计实现结果：
- 长流程任务可控，不会因单次上下文过长失稳。

### [x] 4.3 实现页级结果缓存
目标：
- 按文档、页码、提示词版本、模型配置保存讲解结果。
- 避免重复请求同一页。

预计实现结果：
- 已解析页面可直接读取。
- 重复成本显著降低。

### [x] 4.4 实现单页重跑与局部更新
目标：
- 支持仅重新解析某一页。
- 支持更新后覆盖该页讲解结果。

预计实现结果：
- 用户可迭代修正讲解，不必全书重跑。

### [x] 4.5 实现页级附加提示词
目标：
- 在全局提示词外，允许某一页附加专属要求。

预计实现结果：
- 对公式密集页、图表页、难页可以单独增强讲解。

### [x] 4.6 实现上下文连贯策略
目标：
- 在解析当前页时，引入前文摘要或前几页关键信息。
- 避免逐页讲解彼此割裂。

预计实现结果：
- AI 讲解在跨页时更连贯。

## 阶段 5：学习界面与结果编辑

### [x] 5.1 右侧讲解面板展示优化
目标：
- 右侧展示结构清晰的讲解内容。
- 适配长文本滚动阅读。

预计实现结果：
- 左侧看原 PDF，右侧看 AI 讲解，形成稳定学习界面。

### [x] 5.2 结果手动编辑与保存
目标：
- 支持对 AI 生成结果进行手动修改。
- 保存修改后的最终讲解内容。

预计实现结果：
- 用户可以人工纠正错误或补充个人理解。

### [x] 5.3 当前页状态联动
目标：
- 翻页时右侧自动切换到对应页讲解。
- 未解析页显示明确状态。

预计实现结果：
- 阅读与讲解保持同步，不需要手动查找。

## 阶段 6：导出能力

### [x] 6.1 实现讲解结果导出为项目数据
目标：
- 先把讲解内容稳定导出为 `JSON` 或数据库快照。

预计实现结果：
- 项目结果可备份、可迁移、可恢复。

### [x] 6.2 实现 HTML 预导出
目标：
- 先把“左原页右讲解”的版式导出为 HTML。
- 为最终 PDF 导出验证排版逻辑。

预计实现结果：
- 可快速检查双栏排版是否合理。

### [x] 6.3 实现 PDF 双栏导出
目标：
- 导出“左原页，右讲解”的学习版 PDF。
- 借鉴 `PDF2ZH/PDFMathTranslate` 的双栏导出思路。

预计实现结果：
- 生成可分享、可打印、可长期保存的学习 PDF。

### [x] 6.4 处理长文本分页
目标：
- 当右侧讲解放不下时，自动缩放或自动续页。
- 避免文字溢出边界。

预计实现结果：
- 导出的 PDF 排版稳定可读。

## 协作约定

后续每次开工建议先做这几步：

1. 阅读本文件，确认当前未完成步骤。
2. 由用户指定“开始做哪一步”。
3. 实现后先汇报结果，不自动勾选完成。
4. 只有用户明确确认“该步骤完成”后，才把对应任务从 `[ ]` 改为 `[x]`。
